### Case 3: EvanWritesX — 免手机号注册 Google

|------|------|
| 曝光 | 689K |
| 点赞 | 371 |
| 转发 | 749 |
| 收藏 | 2,200 |

**为什么爆**：
- 解决一个「人人都卡过」的痛点
- 「我刚实测」——真实、新鲜、可复现
- 每一步精确到「蓝色跳过按钮」

### 三条的共同 DNA
1. **解决真实痛点**——不制造需求，满足已有需求
2. **个人真实经历**——不是教程翻译，是亲历者叙述
3. **超高信息密度**——每句话都在给价值，不水
4. **可复现**——用户看完就能做，收藏率极高
5. **反权威表达**——不用术语，用口语、类比、截图
6. **免费给足价值**——正文就是完整教程，变现放在自然位置

## 二、各平台流量机制

### Twitter / X

**算法核心**（来自开源 X 算法逆向）：
- 回复权重 >> 点赞和转发
- 前 30-60 分钟决定生死
- 圈子内互动权重高于陌生人
- 「不感兴趣」信号严重降权
- 原创 >> 转发

**爆款公式**：
```
钩子（反常识 or 具体数字） 
→ 痛点共鸣（"你也遇到过吧"）
→ 解决方案（步骤化）
→ 个人洞察（不是泛泛而谈）
→ 行动号召（收藏/回复）
```

**发布策略**：
- 长推文用 Thread 格式（7-12 条串联）
- 第一条最关键的 5 个钩子类型：反直觉开头、具体结果+悬念、自白+转折、数字承诺、提问
- 在用户活跃时段发布
- 发布后 1 小时内回复每条评论

### 小红书

**CES 评分机制**（2026 更新）：
- 发布后 2 小时内互动占流量分配权重 60%
- AI 内容强制标注
- 点赞率、收藏率、评论深度（字数）是关键指标

**爆款标题 5 公式**：
1. 痛点式：`{痛点}？{方案}`
2. 提问式：`有没有那种{功能}的{产品}？`
3. 发现式：`我发现了个宝藏！{价值}`
4. 热点词：`{热点}爆火后，我用它做了{场景}`
5. 身份共鸣：`{身份标签}必备！{功能}`

**封面铁律**：
- 0.3 秒决定滑走还是点击
- 封面文字 ≤15 字
- 三种高效封面：对比型、大字报型、场景化型
- 80% 点击 = 标题 + 封面组合

**发布节奏**：
- 早 8:00-9:00（曝光高峰）
- 晚 20:00-21:00（活跃高峰）
- 标签：3 流量大词 + 3 垂直词 + 2 场景词

### 知乎

**威尔逊算法**：
- 小流量池冷启动（~100 人）→ 高互动 → 推入更大池
- 回答先推给订阅话题的用户
- 点赞即传播到点赞者主页信息流
- 2025 年转向「深度价值识别模型」
- 长尾推荐周期延长到 3-6 个月

**高赞回答结构**：
```
① 认知冲突开头（100-200 字）
② 痛点共鸣
③ 方法论/方案（2-4 个观点 + 数据/对比）
④ 实操指南（可直接复制使用）
⑤ 结尾升华（金句）
```

**话题策略**：
- 找百度 SEO 高排位关键词对应的问题
- 进入「开源」「GitHub」「程序员」等话题
- 盯热榜，热点事件快速响应
- 前 3 个月发布 50+ 篇干货建立账号标签

## 三、Cascade 三平台内容策略

### 定位矩阵

|  | Twitter/X | 小红书 | 知乎 |
|--|-----------|--------|------|
| **人设** | 独立开发者 + 务实极客 | AI 工具探索者 + 省钱达人 | 开源作者 + 技术分享者 |
| **内容调性** | 英文为主 + 短句 + 观点鲜明 | 中文口语 + 情绪化 + 素人感 | 深度技术 + 真诚分享 + 教程体 |
| **核心价值** | 技术洞察 + 工具推荐 | 省钱技巧 + 效率提升 | 架构设计 + 踩坑复盘 |
| **发布频率** | 每日 1-3 条 | 每周 3-5 篇笔记 | 每周 2-3 篇回答 |
| **变现路径** | GitHub Star → 社区 → SaaS | 工具推荐 → 私域 → 咨询 | 专业影响力 → 付费咨询 → 企业服务 |

### Twitter/X 内容线

**第一条推（发布帖）**：
```
I built a thing.

Claude Code and Codex CLI are great. But Claude API is expensive.
DeepSeek V4 Pro is $0.28/M tokens. The problem? AI coding tools speak "Responses API" and Chinese models only speak "Chat Completions."

So I built a proxy. Zero npm deps. One command. Browser admin panel.

Codex CLI ✓ Claude Code ✓ Cursor ✓ Cline ✓
DeepSeek ✓ Qwen ✓ Kimi ✓ GLM ✓

MIT. github.com/young-ds1/cascade
```

**后续内容线**：
- 「Why I stopped paying $200/month for Claude API」（省钱叙事）
- 「My WebSocket reverse-engineering journey」（技术故事）
- 「How Cascade handles 100 requests/minute on a $5 VPS」（性能展示）
- 「5 things I learned building an AI proxy from scratch」（经验复盘）
- 「Comparing DeepSeek V4 Pro vs Claude Opus for daily coding」（实测对比）

### 小红书内容线

**笔记 1：钩子帖**
- 封面：大字报「别再给 Claude 交税了」
- 标题：独立开发者终于可以用 DeepSeek 写代码了🤖
- 内容：痛点 → 发现 Cascade → 怎么用 → GitHub 链接
- 标签：#AI编程 #独立开发者 #DeepSeek #省钱技巧 #开源工具

**笔记 2：干货教程**
- 封面：终端截图 + emoji 装饰
- 标题：一行命令，让 Codex CLI 白嫖 DeepSeek
- 内容：步骤截图 × 6 + 每步一句话
- 标签：#CodexCLI #AI工具 #效率提升 #程序员日常

**笔记 3：踩坑复盘**
- 封面：红黑对比「我踩了 5 个坑才跑通这个代理」
- 标题：搞了 3 天才让 Claude Code 吃上国产模型的血泪史
- 内容：5 个坑 + 解决方案 + 最后成果
- 标签：#踩坑 #ClaudeCode #DeepSeek #技术分享

**笔记 4：对比测评**
- 封面：四宫格「DeepSeek vs Qwen vs Kimi vs GLM 写代码对比」
- 标题：4 家国产模型写代码，谁最强？（真实实测）
- 内容：同一任务 × 4 模型 → 速度/质量/成本对比表
- 标签：#AI模型对比 #编程效率 #大模型测评

### 知乎内容线

**回答 1：「2025 年了，国内开发者怎么用国产大模型写代码？」**
- 第一部分：国内开发者的真实困境（API 贵、协议不兼容、代理难配）
- 第二部分：现有方案对比（one-api、CCR、OmniRoute 的定位和局限）
- 第三部分：Cascade 的设计思路（协议翻译 + 管理面板 + 零依赖）
- 第四部分：技术细节（WebSocket 帧处理、SSE 流式翻译、供应商适配）
- 第五部分：快速上手 + GitHub 链接
- 风格：坦诚、技术但不炫技、承认不足

**回答 2：「GitHub 上有哪些值得推荐的开源 AI 工具？」**
- 先推别人的优质项目（建立客观性）
- 再自然带入 Cascade

**专栏文章**（如果找不到合适问题）：
- 「从 382 行到 337 行：一个 AI 代理的演进史」
- 「零依赖 Node.js 实现 OpenAI Responses API → Chat Completions 翻译」

## 四、发布作战时间线

### Week 1：冷启动
| 天 | 平台 | 动作 |
|----|------|------|
| Day 1 | Twitter | 发布 Cascade 主帖（Thread 5 条） |
| Day 1 | 小红书 | 发布笔记 1（钩子帖） |
| Day 2 | 知乎 | 发布回答 1 |
| Day 2 | Twitter | 回复主帖所有评论，发第二条（省钱叙事） |
| Day 3 | 小红书 | 发布笔记 2（干货教程） |
| Day 4 | Twitter | 发第三条（技术故事） |
| Day 5 | 小红书 | 发布笔记 3（踩坑复盘） |
| Day 6 | 知乎 | 发布回答 2 |
| Day 7 | 全平台 | 复盘数据，调整 Week 2 策略 |

### 关键原则
1. **发布后 1 小时内**死守评论区，每条回复都认真回
2. **不要一天发多平台**同内容，错开至少 2 天
3. **前 30 条笔记**是和平台互相打标签的阶段，坚持优化
4. **同一个内容核**在不同平台用不同语言和格式表达
5. **宁可少发，不要水**——每条都要有独立价值

## 六、竞品监控

### 需要持续关注的项目
| 项目 | Stars | 定位 | Cascade 差异 |
|------|-------|------|-------------|
| one-api | 33K | Key 分发管理 | 不做协议翻译 |
| CCR (claude-code-router) | 18K | Claude Code 翻译 | 单工具，我们支持 4 工具 |
| openai-reverse-proxy | — | 通用反代 | 无管理面板，无中文 |
| lobe-chat | 60K+ | Chat 客户端 | 不是编程工具代理 |

### 差异化护城河
1. **4 工具 × 4 模型矩阵**——覆盖面最完整
2. **零依赖单文件**——部署成本最低
3. **浏览器管理面板**——唯一可视化方案
4. **中文优先**——文档和面板全中文
5. **MIT 协议**——最宽松
