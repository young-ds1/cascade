# Twitter/X 平台知识库

## 算法核心（来自开源 X 算法逆向工程）

1. **回复权重 >> 点赞和转发**。引发讨论的帖子获得最高分发。
2. **前 30-60 分钟决定生死**。早期互动速度是最关键信号。
3. **圈内互动 > 陌生人互动**。你关注的人的互动权重更高。
4. **「不感兴趣」信号严重降权**。包括：点击不感兴趣、举报、屏蔽。
5. **原创内容 >> 转发**。算法天然压制转发。

## 爆款 Thread 结构

### Phase 1：钩子（5 种验证有效）

1. **反直觉开头**：「Most coding advice is wrong. Here's what actually works.」
2. **具体结果 + 悬念**：「I went from 0 to 4,200 users. Here's the workflow.」
3. **自白 + 转折**：「I spent 18 months building and earned zero. Then I changed 3 things.」
4. **数字承诺**：「5 AI coding mistakes I made (and how I fixed them)」
5. **提问钩子**：「Why do most AI dev tools fail? It's not the tech.」

### Phase 2：内容弧线（7-12 条）
- Tweet 2-3：建立共鸣，让读者觉得「我也是」
- Tweet 4-8：展开论点，每条一个洞察 + 证据
- Tweet 9-10：兑现钩子的承诺，给足价值

### Phase 3：分发
- 发布后 1 小时内回复所有评论
- 跨平台分发（LinkedIn、Threads）带来额外 30-60% 曝光
- 在不同时区窗口自动转推

## 10 条互动法则

1. **具体数字** — 金额、天数、数据抓眼球
2. **模式中断** — 格式翻转打破滚动习惯
3. **负面偏见** — 负面表述更难忘
4. **群体喊话** — 「If you're a developer...」
5. **痛点喊话** — 直击具体痛苦
6. **潜在收益** — 关注转化和奖励
7. **社会证明** — 里程碑、数字、结果
8. **信心和确信** — 说绝对的话，不模糊
9. **主动语态** — 行动导向
10. **警告和规避建议** — 帮人避坑

## Dev Tool 发布模式（2025-2026）

### 已验证公式
> 「I built [tool] to solve [my own pain]. It's open source. Here's how.」

### 成功要素
- 截图/GIF 展示产品实际运行
- GitHub 链接放在第一条回复（非主帖，保持干净）
- CTA：「Star it if you find it useful!」或「PRs welcome!」
- 个人脆弱感：「I'm a solo dev, this took X hours」
- 挑战常识的钩子

### 真实案例数据
- demo 视频 > 截图 > 纯文字（250K views 案例用的是录屏）
- 发布前准备好 landing page（案例中丢失了数小时 signup 窗口）
- 基础准备好再发帖（waitlist、repo、文档）

## 常见错误

- 弱钩子，没有好奇心缺口
- 把 5 条能说完的内容撑到 12 条（导致掉队）
- 发布后没有跨平台分发
- 花 30 分钟写正文，只花 5 分钟写钩子（应该反过来）
- 听起来像 AI 写的营销文案

## 关键发布时机

- 工作日用户活跃时段
- 避免周末深夜
- 大新闻/发布会后蹭热度窗口期 24-48 小时
