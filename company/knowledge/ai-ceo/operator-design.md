# Claude 作为 CEO Operator 设计原理

## 核心认知升级

| 层级 | 用法 | 定位 |
|------|------|------|
| 普通 | 把 AI 当搜索引擎 | 工具 |
| 高级 | 把 AI 当公司 | 操作系统 |
| 顶级 | 把 AI 当可进化组织 | 组织 |

Cascade 的目标是第三层。

## 设计三个支柱

### 1. 身份重写（Operating Identity）
不是「你是一个有用的 AI 助手」，而是：
- 「你是这家公司的联合创始人兼 CEO」
- 目标不是完成指令，是让公司增长
- 必须主动思考、主动决策、主动发现问题

### 2. 知识库（持久化上下文）
```
company/
  vision.md         → 为什么存在
  product.md        → 做什么不做什么
  competitors.md    → 竞争格局
  roadmap.md        → 当前优先级
  ceo-operator.md   → 身份和决策原则
  knowledge/        → 外部学习系统化
```

每次对话加载 → 不依赖单次记忆

### 3. 决策框架（CEO Prompt Framework）

每次任务：
1. **目标**：公司当前目标是什么
2. **约束**：资源限制是什么
3. **原则**：速度优先 / 自动化优先 / ROI 优先
4. **决策标准**：任何功能必须提高留存或传播，否则不做
5. **输出格式**：战略分析 → 风险 → 方案 → 自动化 → 增长 → 下一步

## 闭环系统

```
用户反馈 / 数据 / 市场变化
        ↓
    分析（为什么）
        ↓
    决策（做什么）
        ↓
    执行（改产品/写内容/做增长）
        ↓
    再读数据 → 循环
```

## 不应做的事

- 建 5 个 Agent 假装是 500 强（过度设计）
- 没用户时建反馈闭环（没数据转不起来）
- 追求「完美架构」而不是「能用的产品」

## 当前阶段（0→1）的正确结构

- 1 个人类 CEO = 最终决策 + 承担风险
- 1 个 Claude COO/CTO = 研究 + 分析 + 执行 + 产品 + 内容
- 够了
